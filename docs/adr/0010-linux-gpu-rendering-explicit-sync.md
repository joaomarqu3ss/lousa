# 0010 — Keep WebKitGTK GPU rendering on Linux; disable NVIDIA explicit sync instead

Lousa originally shipped with `WEBKIT_DISABLE_DMABUF_RENDERER=1` on Linux to stop the webview crashing at window creation on Wayland + NVIDIA (GDK "Error 71", tauri-apps/tauri#9304). That workaround forced WebKitGTK into CPU rasterization, which made freehand drawing visibly slower than Excalidraw running in a browser. The crash's actual trigger is the NVIDIA driver's Wayland explicit-sync protocol, so we now set `__NV_DISABLE_EXPLICIT_SYNC=1` instead — verified on Fedora 44 / RTX 3050 / driver 595.80 / WebKitGTK 2.52.4: no crash, and `nvidia-smi` shows the WebKitWebProcess rendering on the GPU. The variable is read only by the NVIDIA driver, so it is inert on Intel/AMD systems, and both variables respect explicit user overrides — a setup that still crashes can fall back with `WEBKIT_DISABLE_DMABUF_RENDERER=1`.

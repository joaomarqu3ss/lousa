# A Workspace of files supersedes single-document-per-window

Lousa opens a **Workspace** — a folder on disk — and surfaces the files it understands (`.excalidraw` Documents and `.md` Notes) through a minimizable sidebar, opening them in **tabs**. This supersedes the v1 "single Document per window" scope. Notes are plain Markdown files: the file on disk is the source of truth, never transformed into the `.excalidraw` format (ADR-0004 stays the Canvas format only). A Note is edited with an **inline live-preview editor built on CodeMirror 6**, where the Markdown text remains the document model — no rich-tree serialization — so the bytes on disk are exactly what the user wrote. That invariant is the spine of the whole design: it keeps Notes portable (they render on GitHub, open in any editor, and are read straight from disk by the user's own agent). The Canvas gains **Links**: a Canvas element can point at a Note (`element.link`), and Notes reference each other with standard relative Markdown links; Lousa reads both as a graph. That graph is the one thing the raw files do not carry, so the Agent Bridge (ADR-0011) exposes it — a division of labor with the agent, which reads Note *content* from disk itself. The Bridge keeps full read/write authority (create Notes, author Links) made safe by **consent**, not by withholding capability: the Checkpoint (ADR-0011) widens beyond the Canvas to snapshot any Note files a turn touches, so "Revert AI changes" undoes canvas and files atomically. Rust continues to own all file I/O (ADR-0003); the sidebar walks the folder lazily behind a curated ignore list, so opening a large repo (an Angular/.NET monorepo, say) stays fast and shows only Notes and Documents, not `node_modules`.

## Considered options

- **Notes embedded in the `.excalidraw` Document** (Markdown in element `customData`) — keeps one file per window and the Bridge reads notes for free, but forces Markdown into a Lousa-proprietary container, breaking portability (no GitHub render, no external editor, no direct agent read). Rejected for plain `.md` files.
- **Notes rendered as live cards on the Canvas** (Obsidian-Canvas style) — the most on-thesis option ("the Canvas is the floor"), but it fights Excalidraw's element and renderer model (see ADR-0002) and adds file-watching plus per-card editing. Deferred as later upside ("pin Note to Canvas"); for now a Note on the Canvas is a *linkable element*, not a rendered card.
- **Rich-tree WYSIWYG editor** (ProseMirror / TipTap / Milkdown) — reaches WYSIWYG faster, but makes the file a lossy serialization of a node tree, so on-disk bytes drift from what the user typed. Rejected because "the file is the source of truth" is the invariant everything else depends on.
- **A second, file-based Bridge transport that re-serves Note content over MCP** — duplicates the agent's own filesystem access and creates two readers/writers of one file. Rejected: the Bridge supplies *structure* (the Link graph); the agent supplies *content*.

## Consequences

- Single-document-per-window is superseded. Lousa manages many open files in tabs, each with its own dirty state; the native close-guard generalizes to every dirty tab.
- New first-class domain terms — **Workspace**, **Note**, **Link** — join Canvas and Document in `CONTEXT.md`.
- The inline live-preview editor (CodeMirror 6) is the largest build in this line and the project's main schedule risk. It is built **split-pane first**, then grown into inline decorations, and ships a lean construct set (inline tables, math, and mermaid deferred).
- The Checkpoint's scope widens from Canvas to Canvas-plus-touched-files, keeping the Bridge's undo a single honest promise.
- `.gitignore`-awareness and a user-editable ignore list are deferred; a curated default ignore list is the v1 safety net against large repos.
- `.excalidraw` remains the Canvas Document format (ADR-0004) and `.md` is the Note format; Lousa reads and writes both but transforms neither into the other.
